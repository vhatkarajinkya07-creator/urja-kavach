# Urja Kavach 🛡️⚡
> **Energy Security & Geopolitical Supply Chain Risk Intelligence Command Center**

Urja Kavach is an end-to-end digital twin and real-time risk intelligence platform built to monitor, simulate, and mitigate geopolitical supply chain shocks on vital energy corridors (e.g., Strait of Hormuz, Red Sea / Bab-el-Mandeb).

---

## ✨ Features
 
- 🚨 **Risk Intelligence Radar**: Real-time risk scoring, tension monitoring, and signal detection across major maritime choke points.
- 🌐 **Interactive Digital Twin**: Interactive geospatial tracking of crude oil shipping routes, naval activity, and risk zones powered by Leaflet.
- 🧪 **Scenario Simulation Engine**: Computes macro impact models (refinery run rates, fuel price surges, GDP impact, power grid stress) during corridor blockades or disruptions.
- 🚢 **Rerouting & Procurement Optimizer**: Automated route cost/delay analysis, supplier risk indexing, and strategic alternative sourcing.
- 🛢️ **Strategic Reserve Buffer Tracker**: Dynamic calculation of buffer days, inventory drawdown rates, and emergency response triggers.
- 📋 **Automated Response Log**: Incident tracking and strategic protocol execution history.

---

## 🛠️ Tech Stack

### Frontend
- **Framework**: React 19 + Vite
- **UI & Styling**: Tailwind CSS, PrimeReact, Lucide Icons
- **Mapping & Viz**: Leaflet, React-Leaflet, Chart.js, Recharts
- **State Management**: Zustand
- **Router**: React Router DOM v7

### Backend
- **Runtime**: Node.js + Express
- **Database**: MongoDB (In-Memory Server / Mongoose ORM)
- **Architecture**: REST API with real-time risk scoring models & scenario engines

## 🚀 Quick Start

### Prerequisites
- Node.js (v18+)
- npm

### 1. Clone Repository
```bash
git clone https://github.com/vhatkarajinkya07-creator/urja-kavach.git
cd urja-kavach
```

### 2. Run Backend
```bash
cd backend
npm install
npm start
```
*Backend runs on `http://localhost:5000` with auto-seeded mock intelligence data.*

### 3. Run Frontend
```bash
cd ../frontend
npm install
npm run dev
```
*Frontend runs on `http://localhost:5173`.*
